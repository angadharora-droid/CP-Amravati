# Hotel Centre Point — Amravati

The website for the Amravati property, plus the staff admin behind it.

The site is built around one idea: **the hotel told as a single day.** You scroll from
06:00 to midnight, the background colour warms and then darkens with the hour, and each
section is the part of the hotel that matters at that time — the kitchens at 08:00, the
rooms at noon, the pool at four, the halls at seven, the bar at eleven.

It is a brochure site with an enquiry form **and a booking engine** — guests check
live availability, pick a room, and pay online through Razorpay (or reserve to pay at
the desk when Razorpay keys aren't set). Staff run the whole thing from **The Desk**:
bookings, room types, rates, and date-level close-outs / rate overrides.

## Layout

```
frontend/          Vite (vanilla JS) → deploys to Vercel
  index.html         the website
  admin.html         the staff inbox
  src/main.js        clock, ambient wash, rooms rail, venue picker, enquiry form
  src/admin.js       login + enquiry inbox
  src/styles.css     the site
  src/admin.css      the admin

backend/           Node + Express + Mongoose → deploys to Render
  src/server.js      app entry
  src/models/        Enquiry, Admin
  src/routes/        /api/enquiries, /api/auth
  src/scripts/       seedAdmin.js
```

Database is **MongoDB**, running locally in development.

## Running it locally

You need Node 18+ and MongoDB running on `127.0.0.1:27017`.

**1. Backend**

```bash
cd backend
npm install
cp .env.example .env        # then edit it — see below
npm run seed:admin          # creates the admin account
npm run dev                 # → http://localhost:5000
```

In `.env`, set `JWT_SECRET` to something long and random:

```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```

and set `ADMIN_EMAIL` / `ADMIN_PASSWORD` to whatever the front desk should use.

**2. Frontend**

```bash
cd frontend
npm install
cp .env.example .env        # VITE_API_URL=http://localhost:5000
npm run dev                 # → http://localhost:5173
```

The website is at `/`. The admin is at `/admin.html`.

## The API

| Method | Route | Who |
| --- | --- | --- |
| `POST` | `/api/enquiries` | public — the form on the site (rate limited to 8 per 15 min) |
| `GET` | `/api/enquiries?status=` | admin |
| `PATCH` | `/api/enquiries/:id` | admin — set status (`new` / `replied` / `archived`) or a note |
| `DELETE` | `/api/enquiries/:id` | admin |
| `POST` | `/api/auth/login` | public — returns a 12h JWT |
| `GET` | `/api/auth/me` | admin |
| `GET` | `/api/rooms/public?checkIn=&checkOut=` | public — room types + availability & price for a stay (room numbers are never sent) |
| `POST` | `/api/bookings/quote` | public — price a stay |
| `POST` | `/api/bookings` | public — create a booking (+ Razorpay order when live) |
| `POST` | `/api/bookings/verify` | public — confirm a Razorpay payment |
| `GET`/`POST`/`PATCH`/`DELETE` | `/api/rooms` | admin — manage room types, rates, inventory, room numbers |
| `GET`/`PATCH`/`DELETE` | `/api/bookings` | admin — booking list + lifecycle (confirm / check-in / check-out / cancel) |
| `GET`/`POST`/`DELETE` | `/api/blocks` | admin — close-outs & seasonal rate overrides |
| `GET` | `/health` | anyone |

**Payments.** Set `RAZORPAY_KEY_ID` and `RAZORPAY_KEY_SECRET` in the backend `.env`
to take online payments. Leave them empty and the engine still works — bookings are
confirmed as "pay at the hotel desk." The browser gets the public key from the
create-booking response, so there is no Razorpay key to set on the frontend.

**Inventory.** Room types are seeded once on first boot (`src/scripts/seedRooms.js`,
also `npm run seed:rooms`): 62 currently-sellable rooms across Executive, Premium,
Family Premium, Club and Deluxe; the Luxury Suite ships inactive (held offline).
Everything after that is editable in The Desk.

## Deploying

**Backend → Render.** `backend/render.yaml` describes the service. Point Render at the repo;
it will pick up the blueprint. Then set two environment variables in the dashboard:

- `MONGO_URI` — your hosted MongoDB connection string (Render cannot see the Mongo on your
  laptop, so this needs to be an Atlas cluster or similar in production)
- `CORS_ORIGINS` — your Vercel URL, e.g. `https://centrepointamravati.vercel.app`

`JWT_SECRET` is generated by Render automatically. After the first deploy, run the admin
seed once from the Render shell: `npm run seed:admin`.

**Frontend → Vercel.** Set the root directory to `frontend`. Vercel reads `vercel.json` and
runs `npm run build`. Add one environment variable:

- `VITE_API_URL` — your Render URL, e.g. `https://centrepoint-amravati-api.onrender.com`

Vite bakes that in at build time, so redeploy the frontend after changing it.

## Photography

Every image on the site is a styled placeholder — a plum-and-sand gradient, not a photo.
They are marked with `data-photo="..."` in the HTML. To drop real photographs in, add a rule
per plate in `src/styles.css`:

```css
[data-photo='pool']   { background-image: url('/images/pool.jpg'); }
[data-photo='facade'] { background-image: url('/images/facade.jpg'); }
```

Put the files in `frontend/public/images/`. The rooms rail uses the same attribute.

## A note on the content

The copy, room names, rates, phone numbers and addresses in here are **placeholders written
for this build**. Replace them with the real details before this goes anywhere near the
